# 2-4 160 Hour Practice Plan

## Total Effort

160 hours for one student.

## Weekly Breakdown

### Week 1 - 40 hours

Objective: Project setup, repository creation, requirements review, wireframes, database draft, Docker baseline.

Required GitHub evidence:

- Weekly report file committed in 06_weekly_reports.
- Source code commits from the week.
- Screenshots or terminal output if relevant.
- Updated project board or issue list.
- Short summary of problems and solutions.

### Week 2 - 40 hours

Objective: Core database implementation, backend structure, first API endpoints, frontend skeleton, initial test cases.

Required GitHub evidence:

- Weekly report file committed in 06_weekly_reports.
- Source code commits from the week.
- Screenshots or terminal output if relevant.
- Updated project board or issue list.
- Short summary of problems and solutions.

### Week 3 - 40 hours

Objective: Main feature implementation, validation, authentication if required, UI improvements, integration testing.

Required GitHub evidence:

- Weekly report file committed in 06_weekly_reports.
- Source code commits from the week.
- Screenshots or terminal output if relevant.
- Updated project board or issue list.
- Short summary of problems and solutions.

### Week 4 - 40 hours

Objective: Testing, bug fixing, final documentation, screenshots, final report, GitHub cleanup, presentation preparation.

Required GitHub evidence:

- Weekly report file committed in 06_weekly_reports.
- Source code commits from the week.
- Screenshots or terminal output if relevant.
- Updated project board or issue list.
- Short summary of problems and solutions.

## Daily Work Expectation

The student should work in small commits. Every major change must be committed with a clear message. Examples:

- add initial database schema
- implement user list page
- add Docker Compose configuration
- fix validation for required fields
- update weekly report

## Weekly Review Rule

At the end of every week the student must push all changes to GitHub and submit the weekly report link to the practice supervisor.
